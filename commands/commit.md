Create git commits from current staged and unstaged changes.

**Philosophy:** Prefer multiple focused commits over one large commit. Each commit should have a single scope/purpose (e.g., separate commits for docs, refactoring, bug fixes, features).

**Arguments:** $ARGUMENTS

If arguments are provided, use them as additional context or constraints for the commit. Examples:
- `/commit only bugfix` → Only commit bug fix changes, leave others unstaged
- `/commit docs only` → Only commit documentation changes
- `/commit single` → Create a single commit for all changes instead of splitting
- `/commit -m "custom message"` → Use the provided message for the commit

Instructions:
1. Run `git status` and `git log --oneline -5` (as separate Bash calls, never chain with `&&` — permission patterns match command prefix only)
3. Analyze changes and group them by scope/purpose
4. For each logical group:
   a. Run `git diff` on specific files to review changes
   b. Stage only related files with `git add <files>`
   c. Create a commit with a clear, scoped message
   d. Report: files committed and commit message
5. Repeat step 4 until all changes are committed
6. Run final `git status` to confirm clean working tree
7. Report summary: list of commits created

Rules:
- Do NOT include any Claude watermark or "Generated by" text
- Do NOT push to remote unless explicitly asked
- Group by: docs, feat, fix, refactor, chore, test, style, ci
- Single-file changes can be committed individually
- Related multi-file changes should be one commit
